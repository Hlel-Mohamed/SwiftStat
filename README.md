# SwiftStat

Fast D&D rules reference — search, voice, and instant stat cards. Built as an
offline-ready **PWA** (installable to a phone home screen without an app store).
Supports two rules editions: **5e (2014, SRD 5.1)** and **5.5 (2024, SRD 5.2.1)**.

## Features

- **Instant search** over spells, conditions, monsters, equipment, magic items, feats,
  and more — client-side and offline. Monster trait/action text is indexed, so searching
  `multiattack`, `pack tactics`, or `finesse` finds the right cards.
- **Attack math** without an LLM: type `rogue level 5 18 dex daggers full attack` →
  `4 + 2d4 + 3d6` (incl. Sneak Attack). Deterministic — computed from SRD rules, not guessed.
  Give the ability as a **score** (`18 dex`) or a **modifier** (`+4 dex`); a bare weapon
  name just shows the item card (no spurious calc).
- **Edition toggle** (5e / 5.5), persisted. The 2024 SRD ships no spells/monsters, so those
  are shown from 5.1 with a small "5.1" badge.
- **Tap-to-talk voice** search (Web Speech API where supported).
- **Works offline** and installs to your home screen (service worker + web manifest).

## Run it

```bash
npm install
npm run data     # generate public/data/srd-*.json from a sibling ../5e-database clone
npm run dev      # http://localhost:5173
npm run build    # production build into dist/
npm run preview  # serve the built app
npm test         # unit tests (Vitest) for the engine
npm run lint     # oxlint
```

## Architecture

Two independent halves, on purpose:

1. **The index** (`public/data/srd-*.json` + `src/engine/search.js`) — the SRD as fetched,
   service-worker-cached JSON (not bundled into the app), indexed with MiniSearch. Data is
   generated by `scripts/build-data.mjs`; **it is not hand-edited**.
2. **The calculator** (`src/engine/dice.js`, `src/engine/query.js`) — a rule-based parser
   + dice engine. No AI, so it's fast, offline, and accurate.

## Data

`public/data/srd-*.json` is **generated** by `npm run data` from a sibling clone of
[5e-bits/5e-database](https://github.com/5e-bits/5e-database) at `../5e-database`, then
committed (CI can't regenerate it). After the upstream data changes, re-run `npm run data`
and commit the result.

## Build profiles (public vs personal)

`SWIFTSTAT_PROFILE` (default `public`) controls what ships:

- **public** — SRD-only data; safe to host. This is what the GitHub Pages deploy builds.
- **personal** — also merges your git-ignored custom cards from `personal-data/` (see
  `personal-data/README.md`), shown with a "Custom" badge. For local / private use only.

```bash
npm run data:personal   # merge personal-data/ into git-ignored personal-*.json
npm run dev:personal
npm run build:personal
```

## Deploy

- **GitHub Pages:** push to `main`; `.github/workflows/deploy.yml` builds the **public**
  profile with `BASE_PATH=/SwiftStat/` and publishes. Enable Pages → Source: GitHub Actions.
  Data is intentionally pre-committed (CI has no `../5e-database`).
- **Vercel/Netlify/custom domain:** deploy `dist/` with default `BASE_PATH=/`.
